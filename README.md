# 餐饮门店数字化管理平台

## 项目简介

针对餐饮门店高峰期订单处理效率低、库存易超卖等问题，设计并实现的后端系统。

## 技术栈

- **后端框架**：Java 21 + Spring Boot 3.3.4
- **数据库**：MySQL 8.0.11
- **缓存**：Redis 8.4.0
- **消息队列**：RabbitMQ 3.13.7-management
- **实时通信**：WebSocket + STOMP
- **容器化**：Docker + Docker Compose

## 核心特性

### 1. 高并发下单处理
- 使用 Redis + Lua 脚本实现库存预扣减，保证操作原子性，避免超卖问题
- 本地 1000 个请求并发压测，**QPS 达到 277/sec**，平均响应时间 168ms，错误率 0%

### 2. 异步解耦与消息可靠性
- 基于 RabbitMQ 实现业务异步处理
- 结合事务监听（TransactionalEventListener）确保数据库提交后再发送消息
- 使用手动 ACK + 持久化机制，提高消息可靠性

### 3. 实时通信
- 使用 STOMP 协议实现订单实时推送
- 设计心跳检测与断线重连机制，提高系统稳定性

### 4. 性能优化
- 通过压测定位连接池瓶颈，优化 Redis 连接池（8→50）和 MySQL 连接池（10→30）
- 优化后 **QPS 提升 92%**，响应时间降低 65%

### 5. 规范化开发
- 严格遵循 RESTful API 规范，利用 Knife4j 自动生成动态接口文档
- 设计全局异常处理器（@RestControllerAdvice）与统一结果封装类（Result<T>）
